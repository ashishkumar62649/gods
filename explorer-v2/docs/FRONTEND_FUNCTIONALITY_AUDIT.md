## Immediate Decisions

- Mock data remains allowed only under `frontend/src/data/mock` for development fallback, but production components should prefer backend API clients.
- High-frequency live points use the God Eyes fixed binary format for flights now. Satellites and maritime have backend binary endpoints, but frontend decoders are still pending.
- Heavy geospatial line/polygon layers should use MVT endpoints already exposed by the backend. Current cable rendering uses bounded JSON as a transitional fallback.
- Watch-zone persistence is a real backend/database gap. The zone list/filter UI is locally interactive against `frontend/src/data/mock/watch-zones` until tables and `/api/v2/watch-zones` endpoints exist; active alerts now come from `/api/v2/hazards`.
- Unused mock-only overlay/rendering files were moved to `_migration/pending-review/frontend-mock-ui` so they do not remain in the active production path.
